ArrayList

Introduction

Unlike arrays in Java, ArrayLists have dynamic sizes. This means that they may have an initial set size, but it can be changed depending on the number of elements present in the ArrayList. Also, ArrayLists can only work with non-primitive types such as Integer, String, and other classes which may be created by the user.

Autoboxing/Autowrapping

Processes called autoboxing/autowrapping are useful in ArrayLists as ArrayLists do not use primitive data types. In short, autoboxing converts primitive data types, such as int and double, into their wrapper class form, Integer and Double. Autounboxing is the reverse, converting from wrapper class form to primitive data type. Wrapper classes are Java classes that are intended to hold a single instance of the associated primitive type's data value. When you want to store primitive data types in Java structures that demand objects, like ArrayLists, you utilize them.

Commonly Used Methods for ArrayList

  • - add(int index, E element) | Add a value to the specified index and push the original value and all that follow to the left
  • - set(int index, E element) | Set an index's value to a specified value
  • - get(int index) | Get the value of the index specified
  • - remove(int index) | Remove the element at the index specified
  • - clear() | Remove all elements in the ArrayList
  • - size() | Return the size/length of the ArrayList
  • - isEmpty() | Check if the ArrayList is empty
  • - contains(E element) | Check if the ArrayList contains a certain value
  • - indexOf(E element) | Find the index of a certain value within the ArrayList

Access Elements

Accessing elements in an ArrayList is the exact same as normal arrays in Java. Keep in mind that ArrayList sizes are dynamic, so if you add/remove an element from the ArrayList, the size of the ArrayList will change.
