Recursion

Introduction

Recursion is when a method invokes itself, useful in solving complex problems. This leads to the method being broken down into smaller and more simplified cases until a certain case is reached. The certain case or cases which terminate the method is/are called the base case(s). Since the base case is “terminating” the method, there are no recursive calls to invoke the method again. Base cases are required or else an infinite recursion, also known as stack overflow, will take place. Recursive cases cause the method to call itself and break the method down in subbranches for a “smaller” task.

Mergesort

Mergesort uses recursion to sort elements into a certain order. First, it takes the original array and splits it roughly into two halves. Then, it recursively sorts by repeatedly splitting the arrays into halves, and then sorting them using mergesort. As the name suggests, the arrays then merge together and slowly start to become sorted. In merge sort, there are about n·log2n comparisons, and the time complexity is O(n log n).
